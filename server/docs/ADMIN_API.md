# Admin API Documentation

Base URL (local): `http://localhost:4000`

All admin endpoints return JSON in this pattern:

- Success: `{ "success": true, "data": ... }`
- List success: `{ "success": true, "data": [...], "meta": {...} }`
- Error: `{ "success": false, "error": "message" }`

## Authentication

### Login
- `POST /api/admin/auth/login`
- Public endpoint (no token required)

Request body:
```json
{
  "email": "admin@sabyorder.com",
  "password": "StrongPassword123!"
}
```

Response data:
```json
{
  "token": "jwt-token",
  "admin": {
    "id": "uuid",
    "fullName": "Admin User",
    "email": "admin@sabyorder.com",
    "roles": ["super_admin"]
  }
}
```

Notes:
- Server sets refresh token in HttpOnly cookie: `apsara_admin_rt`.
- Client should send requests with credentials (cookie included).

### Current admin profile
- `GET /api/admin/auth/me`
- Requires header: `Authorization: Bearer <token>`

### Refresh access token
- `POST /api/admin/auth/refresh`
- Public endpoint (uses refresh token from HttpOnly cookie)

Response data:
```json
{
  "token": "new-jwt-token",
  "admin": {
    "id": "uuid",
    "fullName": "Admin User",
    "email": "admin@sabyorder.com",
    "roles": ["super_admin"]
  }
}
```

Notes:
- Refresh token is rotated and re-set in cookie on each refresh.

### Logout
- `POST /api/admin/auth/logout`
- Public endpoint (works with refresh cookie; access token optional)

Response:
```json
{
  "success": true,
  "data": { "loggedOut": true }
}
```

Notes:
- Logout revokes refresh token from cookie.
- If cookie is missing, logout revokes all active refresh tokens of current admin.
- Client should still remove stored access `token` and profile after logout.

## Role Matrix

Roles in system:
- `super_admin`
- `catalog_admin`
- `ops_admin`

Access summary:
- Products, Categories, Variant stock, Image uploads: `super_admin`, `catalog_admin`
- Orders: `super_admin`, `ops_admin`
- Returns: `super_admin`, `ops_admin`
- Analytics: `super_admin`, `catalog_admin`, `ops_admin`
- Audit logs: `super_admin` only

## Products

### List products
- `GET /api/admin/products`
- Roles: `super_admin`, `catalog_admin`

Query params:
- `page` (default `1`)
- `limit` (default `20`, max `100`)
- `q` (title search)
- `gender` (`men|women|unisex`)
- `category_id`
- `is_active` (`true|false|1|0|yes|no`)

### Get product detail
- `GET /api/admin/products/:id`
- Roles: `super_admin`, `catalog_admin`

Returns product with `images` and `variants`.
Includes `product_code` (example: `P-000123`) for human-friendly reference.

### Create product
- `POST /api/admin/products`
- Roles: `super_admin`, `catalog_admin`

Required fields:
- `title`
- `slug`
- `gender` (`men|women|unisex`)
- `category_id` (must reference active category)
- `base_price` (> 0)
- `images` (at least 1)
- `variants` (at least 1)

Optional fields:
- `description`
- `discount_price` (> 0 and `<= base_price`)
- `tags` (object)
- Catalog metadata fields merged into `tags`:
  - `brand`
  - `parent_sku`
  - `material`
  - `care`
  - `fit`
  - `model_note`
  - `size_guide`
  - `style_tags` (array or comma-separated string)
  - `meta_title`
  - `meta_description`

Notes:
- System primary key is still UUID (`id`).
- `product_code` is auto-generated by backend and can be used as a human-readable reference in admin UI.

Example request:
```json
{
  "title": "Classic Oxford Shirt",
  "slug": "classic-oxford-shirt-white",
  "gender": "men",
  "category_id": "f0e4b5e8-9ce4-4d6f-a3ce-2e2eb1f0a4f2",
  "description": "Smart casual cotton shirt.",
  "base_price": 39.99,
  "discount_price": 32.99,
  "brand": "Saby Order",
  "parent_sku": "APS-SHIRT-OXFORD",
  "material": "100% cotton",
  "care": "Machine wash cold",
  "fit": "Regular fit",
  "model_note": "Model wears size M",
  "size_guide": "S-XL regular sizing",
  "style_tags": ["smart-casual", "office"],
  "meta_title": "Classic Oxford Shirt for Men | Saby Order",
  "meta_description": "Breathable cotton oxford shirt for work and weekend styling.",
  "images": [
    {
      "url": "http://localhost:4000/uploads/products/1710000000000-abcd1234.webp",
      "alt_text": "Front view",
      "sort_order": 0
    }
  ],
  "variants": [
    { "size": "M", "color": "White", "sku": "APS-SHIRT-OXFORD-WHT-M", "stock": 25 },
    { "size": "L", "color": "White", "sku": "APS-SHIRT-OXFORD-WHT-L", "stock": 18 }
  ]
}
```

### Update product
- `PATCH /api/admin/products/:id`
- Roles: `super_admin`, `catalog_admin`

Updatable fields:
- `title`, `slug`, `gender`, `category_id`, `description`
- `base_price`, `discount_price`
- `is_active`
- `tags`
- same catalog metadata fields as create (`brand`, `parent_sku`, etc.)
- `images` (replace full list if provided)
- `variants` (replace full list if provided)

### Soft delete product
- `DELETE /api/admin/products/:id`
- Roles: `super_admin`, `catalog_admin`
- Behavior: sets product inactive (`is_active=false`)

### Update variant stock
- `PATCH /api/admin/variants/:id/stock`
- Roles: `super_admin`, `catalog_admin`
- `:id` must be the **variant id** (`product_variants.id`), not product id.

Request body:
```json
{ "stock": 12 }
```

If variant id does not exist, API returns `404` with error `Variant not found`.

## Categories (Product Types)

### List categories
- `GET /api/admin/categories`
- Roles: `super_admin`, `catalog_admin`

### Get category detail
- `GET /api/admin/categories/:id`
- Roles: `super_admin`, `catalog_admin`

### Create category
- `POST /api/admin/categories`
- Roles: `super_admin`, `catalog_admin`

Request body:
```json
{
  "name": "Shirts",
  "slug": "shirts",
  "sort_order": 0,
  "is_active": true
}
```

Notes:
- `name` required
- `slug` auto-slugified from `slug` or `name`
- `sort_order` must be integer `>= 0`

### Update category
- `PATCH /api/admin/categories/:id`
- Roles: `super_admin`, `catalog_admin`

Updatable:
- `name`
- `slug`
- `sort_order`
- `is_active`

### Soft delete category
- `DELETE /api/admin/categories/:id`
- Roles: `super_admin`, `catalog_admin`
- Behavior: sets category inactive (`is_active=false`)

## Image Uploads

### Upload product images
- `POST /api/admin/uploads/images`
- Roles: `super_admin`, `catalog_admin`
- Content-Type: `multipart/form-data`
- Field name: `files` (up to 10 files)

File constraints:
- Allowed mime: `image/jpeg`, `image/png`, `image/webp`
- Max size per file: `5MB`

Response:
```json
{
  "success": true,
  "data": [
    {
      "url": "http://localhost:4000/uploads/products/1710000000000-abcd1234.webp",
      "filename": "1710000000000-abcd1234.webp",
      "size": 124532,
      "mime": "image/webp",
      "provider": "local"
    }
  ]
}
```

Storage behavior:
- Local mode (`UPLOAD_STORAGE=local`): serves files at `/uploads/products/...`
- Cloudinary mode (`UPLOAD_STORAGE=cloudinary`): returns `secure_url`

## Orders

### List orders
- `GET /api/admin/orders`
- Roles: `super_admin`, `ops_admin`

Query params:
- `page` (default `1`)
- `limit` (default `20`, max `100`)
- `status` (`pending|paid|packed|shipped|delivered|cancelled`)
- `q` (search by order code, email, order id)

Response includes:
- `data` (orders)
- `meta` (pagination)
- `statuses` (allowed status list)
- `transitions` (allowed status transition map)

### Get order detail
- `GET /api/admin/orders/:id`
- Roles: `super_admin`, `ops_admin`

Returns order with:
- `items`
- `history`

### Update order status
- `PATCH /api/admin/orders/:id/status`
- Roles: `super_admin`, `ops_admin`

Request body:
```json
{ "status": "packed" }
```

Validation:
- must be one of allowed statuses
- transition must be valid (example: `paid -> packed` allowed, `pending -> shipped` not allowed)

## Returns

### List return requests
- `GET /api/admin/returns`
- Roles: `super_admin`, `ops_admin`

Query params:
- `page` (default `1`)
- `limit` (default `20`, max `100`)
- `status` (`requested|approved|rejected|resolved`)
- `reason` (`damaged|wrong_item|not_as_described|size_issue|other`)
- `q` (search by return id, order code, email)

Response includes:
- `data` (return requests)
- `meta` (pagination)
- `statuses` (allowed status list)
- `transitions` (allowed status transition map)

### Get return request detail
- `GET /api/admin/returns/:id`
- Roles: `super_admin`, `ops_admin`

Returns request with linked `items` from the related order.

### Update return request status
- `PATCH /api/admin/returns/:id/status`
- Roles: `super_admin`, `ops_admin`

Request body:
```json
{ "status": "approved", "note": "Item inspection approved. Return label sent." }
```

Validation:
- must be one of allowed statuses
- `note` is optional except when setting status to `rejected` (required)
- transition must be valid:
  - `requested -> approved|rejected`
  - `approved -> resolved|rejected`
  - `rejected -> (none)`
  - `resolved -> (none)`

## Analytics

All analytics endpoints:
- Base: `/api/admin/analytics`
- Roles: `super_admin`, `catalog_admin`, `ops_admin`

### Overview
- `GET /api/admin/analytics/overview?days=7`
- `days` is clamped to `1..90`

### Top products
- `GET /api/admin/analytics/top-products?days=7&limit=10`
- `days` clamped `1..90`
- `limit` clamped `1..50`

### Top searches
- `GET /api/admin/analytics/top-searches?days=7&limit=10`
- `days` clamped `1..90`
- `limit` clamped `1..50`

## Audit Logs

### List audit logs
- `GET /api/admin/audit-logs`
- Roles: `super_admin`

Query params:
- `page` (default `1`)
- `limit` (default `20`, max `100`)
- `action`
- `entity_type`
- `entity_id` (exact match)
- `actor_admin_id`
- `from` (ISO datetime, inclusive)
- `to` (ISO datetime, inclusive)
- `sort` (`created_at_desc` default, or `created_at_asc`)
- `q` (search action/entity/entity_id/admin name/admin email)

---

## Common HTTP Errors

- `400` validation errors (bad payload, missing required fields, invalid status transition input)
- `401` unauthorized (missing/invalid/expired admin token)
- `403` forbidden (role does not have permission)
- `404` not found
- `409` conflict (duplicate slug/SKU, FK conflicts, invalid status transition)
- `500` server error

## Quick cURL Examples

Login:
```bash
curl -X POST http://localhost:4000/api/admin/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"admin@sabyorder.com","password":"StrongPassword123!"}'
```

List products:
```bash
curl "http://localhost:4000/api/admin/products?page=1&limit=10" \
  -H "Authorization: Bearer <TOKEN>"
```

Create category:
```bash
curl -X POST http://localhost:4000/api/admin/categories \
  -H "Authorization: Bearer <TOKEN>" \
  -H "Content-Type: application/json" \
  -d '{"name":"Jeans","slug":"jeans","sort_order":1}'
```
